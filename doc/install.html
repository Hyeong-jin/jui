Common

Minify

JUI 라이브러리는 스타일만 별도로 사용할 수 있으며 하나의 패키징된 파일만 로드하면 됩니다.
그리고 jui 클래스가 설정되어 있는 마크업의 하위에서만 사용할 수 있으니 이점 참고하시기 바랍니다. 

<link rel="stylesheet" href="jui.min.css" />
<link rel="stylesheet" href="animate.css" />
<body class="jui">
	...
</body>

스크립트는 jQuery 1.8 이상에서만 동작하므로 관련 파일을 먼저 로드해야 합니다. 

<script src="jquery.min.js"></script>
<script src="jui.min.js"></script>

Debug

스크립트는 디버깅 용도로 각각 별도로 로드할 수 있으며 base와 core는 필수 로드 파일입니다. 

<script src="js/base.js"></script>
<script src="js/core.js"></script>

다음으로는 ui 폴더 안의 파일들을 먼저 로드해야 하며 uix의 파일들은 맨 마지막에 로드합니다. 

<script src="js/base.js"></script>
<script src="js/core.js"></script>
<script src="js/ui/button.js"></script>
<script src="js/ui/combo.js"></script>
<script src="js/ui/datepicker.js"></script>
<script src="js/ui/dropdown.js"></script>
<script src="js/ui/modal.js"></script>
<script src="js/ui/notify.js"></script>
<script src="js/ui/paging.js"></script>
<script src="js/ui/tooltip.js"></script>
<script src="js/uix/autocomplete.js"></script>
<script src="js/uix/tab.js"></script>
<script src="js/uix/table.js"></script>
<script src="js/uix/tree.js"></script>
<script src="js/uix/window.js"></script>
<script src="js/uix/xtable.js"></script>

필요에 따라 사용하지 않는 UI 파일을 로드하지 않아도 되며 uix 폴더의 UI는 ui 폴더의 UI를 사용하기 때문에 주의하시기 바랍니다.

Build & Test

테스트를 수행하고 패키지에 포함된 .less와 .js 파일을 하나의 파일로 합치기 위해서는 아래와 같은 명령어를 실행하면 됩니다.
참고로 빌드 도구인 Grunt 모듈은 NodeJS가 설치되어 있어야 합니다. npm install grunt-cli -g
cd jui
npm install
grunt
